Variables, Data Types, and Naming Standards

In JavaScript, we use variables to store values. Think of variables as containers that hold data.

Cup/Bottle/Jar Analogy for Variables

Variables are like containers that hold liquids. These containers can be:

  • var: An older container that can spill.
  • let: A newer, safer container with a lid.
  • const: A sealed container that cannot be changed.

Data Types Analogy

Just as we use specific containers in our kitchen to store certain items, JavaScript has standards for which data type should store which kind of data:

  • Number: a number holds a numeric value (e.g., 5, 3.14).
  • String: a string holds text (e.g., "Hello, world!").
  • Boolean: a boolean holds only two possible values: true or false.
  • Object: Like a bottle filled with cashews, badams, and pistas, an object holds multiple items under one name, with each item having its count (e.g., { cashews: 10, badams: 15, pistas: 20 }).
  • Array: Like an egg tray where each row holds a certain number of eggs, an array holds multiple values in a specific order (e.g., [6, 8, 10] where each number represents the number of eggs in a row).

In the kitchen, we use plastic covers to store only solid items, because liquids can spill. Similarly, JavaScript has standards that certain types of data should be stored in specific data types. For instance, numbers (liquid) go into bottles, while objects and arrays (solids) go into boxes and trays to keep them organized and prevent them from spilling.

Camel Case: A Naming Standard

In JavaScript, variables and functions follow the Camel Case naming convention:

  • Start with a lowercase letter.
  • Capitalize the first letter of each subsequent word.
  • Good variable names: userAge, totalPrice
  • Bad variable names: totalprice (hard to read), 1userName (can’t start with a number), let (reserved keyword).

Code Examples

// Using var (Old Style):
var userAge = 25; // Can be changed, but may cause unpredictable behavior

// Using let (Modern & Safe):
let temperature = 72; // Can be updated later, but only in the same block of code

// Using const (Sealed):
const birthYear = 1995; // Cannot be changed once set

// Camel Case Examples:
let myFavoriteDrink = 'Coffee'; // Camel case variable name
const userAge = 30; // Good variable name
