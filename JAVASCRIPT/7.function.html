Functions and Reusability

Functions are powerful tools in programming. They let you define a single set of instructions that can work with different inputs and produce different results. Think of a function as a *mixer* that takes ingredients (inputs) and creates a unique blend (output) based on what you provide. For example, you can use the same mixer to make different fruit juices by providing different fruits.

Function Definition

Here's how you can define a function that "mixes" fruits into a juice: 

// Define a function to mix fruits
function makeJuice(fruit1, fruit2) {
    return `Here is your juice made of ${fruit1} and ${fruit2}!`;
}

In this case:

  • The function is makeJuice.
  • The inputs (parameters) are fruit1 and fruit2.
  • The output is a message describing the juice made from the provided fruits.

Function Call

To use the function, you call it with specific fruits as arguments. Here's how you can create different juices: 

// Call the function with different fruits
let juice1 = makeJuice("Apple", "Orange");
let juice2 = makeJuice("Banana", "Strawberry");

console.log(juice1); // Output: Here is your juice made of Apple and Orange!
console.log(juice2); // Output: Here is your juice made of Banana and Strawberry!

By changing the input fruits, the same function produces different results.

Why Use Functions?

Functions save time and effort by allowing you to reuse the same logic for different scenarios. Instead of writing the mixing process repeatedly, you define it once and use it with any combination of fruits: 

// Reuse the same function for various combinations
let tropicalJuice = makeJuice("Pineapple", "Coconut");
let citrusJuice = makeJuice("Lemon", "Lime");

console.log(tropicalJuice); // Output: Here is your juice made of Pineapple and Coconut!
console.log(citrusJuice);   // Output: Here is your juice made of Lemon and Lime!

Naming Conventions

Follow these best practices for naming functions to make your code clear and maintainable:

  • Use Verb Phrases: Start function names with an action word that explains what the function does, such as makeJuice, calculateTotal, or fetchData.
  • Use Camel Case: Start with a lowercase letter, and capitalize the first letter of subsequent words. For example: getUserDetails.
  • Be Descriptive: Avoid vague names like doStuff. Clearly state the function’s purpose.
  • Prefix Common Actions: Use prefixes like get (retrieve data), set (update a value), or is/has (return a boolean).
  • Avoid Abbreviations: Write full words for clarity. For instance, use calculateTax instead of calcTax.
// Good Naming Examples
function calculateArea(length, width) {
    return length * width;
}

function isUserLoggedIn(user) {
    return user.isLoggedIn;
}

function fetchProductDetails(productId) {
    return database.getProduct(productId);
}

// Bad Naming Examples
function doStuff() {
    // unclear what this does
}

function calc() {
    // too vague
}
